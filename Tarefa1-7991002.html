class LemmaTokenizer(object):
    def __init__(self):
        self.wnl = WordNetLemmatizer()
    def __call__(self, doc):
        return [self.wnl.lemmatize(t) for t in word_tokenize(doc)]

III - Descrição das técnicas utilizadas¶

A primeira técnica utilizada para realizar a tarefa foi validação cruzada, utilizando o método holdout, que consiste em dividir os dados de treinamento entre dois conjuntos disjuntos: um de treino e um de validação. Isso foi feito através do método train_test_split, com a proporção de dados de treino sendo 2/3. Assim, os parâmetros do modelo são estimados no conjunto de treino, e o erro de predição é obtido a partir dos dados do conjunto de validação. Dessa forma é possível avaliar a capacidade de generalização do modelo. Além disso, o conjunto de validação foi utilizado para reduzir a dimensionalidade dos dados (iterando o número de features, avaliou-se qual produzia maior acurácia). Nesse tarefa, foram considerados como 'dados' apenas os textos dos tweets.

Em seguida, o modelo de saco de palavras foi utilizado para representar os dados. Nesse modelo, não consideramos a posição nem a ordem dos termos. Assim, através da classe CountVectorizer, o texto é representado por um vetor em que cada dimensão corresponde a um token, e o valor da dimensão equivale à quantidade de ocorrências do termo no texto. Uma forma de representação alternativa utilizada na realização da tarefa foi obtida através da classe TfidfVectorizer. A diferença nesse caso é que o valor de cada dimensão equivale ao tf-idf do termo no tweet, que é obtido através da fórmula $tf * idf$ (ou por vezes $tf * (idf+1)$), em que tf é a frequência (número de ocorrências) do termo no tweet idf é o inverso da frequência do documento (inverso do número de tweets em que o termo ocorre).

Depois, foram utilizados alguns classificadores lineares e avaliamos a acurácia de cada um sobre o conjunto de treino e sobre o de validação. Os classificadores utilizados foram:

1) Perceptron: O algoritmo do Perceptron define uma hipótese final $g(x) = sign(w^tx)$. Ele basicamente inicializa um vetor de pesos $w(0)$ arbitrário, e para cada iteração $t \ge 0$, escolhe qualquer dado $(x(t), y(t))$ cuja classificação esteja errada (ou seja, $y(t) \ne sign(w(t)^tx(t))$) e atualiza $w(t+1) = w(t) + y(t)x(t)$. O algoritmo para quando não há mais dados mal classificados. Esse algoritmo sempre termina se os dados são linearmente separáveis.

2) Regressão Linear: Basicamente tenta achar a hipótese $h$ que minimiza o erro quadrático entre $h(x)$ e $y$. Para isso, constrói a matriz X e o vetor y com os dados $(x_1,y_1), (x_2,y_2), \ldots, (x_N,y_N)$ (em que cada $x_i$ é um vetor de dimensão $d$ e $y_i$ corresponde à sua classe), e calcula a pseudo-inversa de X: $X^\dagger = (X^TX)^{-1}X^T$. A partir disso, retorna $w = X^\dagger y$.

3) Regressão Logística: Basicamente, tenta atualiza vetor de pesos $w$ de forma a minimizar $\dfrac{1}{N}\sum_{n=1}^{N}\ln(\dfrac{1}{\theta({y_nw^Tx_n})})$, em que $\theta$ é uma função logística (como por exemplo, $\theta(s) = \frac{e^s}{1+e^s}$). Para isso, pode-se utilizar o algoritmo de Gradiente Descendente, que basicamente tenta "descer" pela surperfície que se deseja minimizar até encontrar um ponto mínimo.

Uma outra técnica que foi abordada nessa tarefa foi a de Stemming, que consiste em uma heurística que corta o fim das palavras em uma tentativa de obter sua raiz. Essa heurística é baseada em algumas regras, como por exemplo, palavras terminadas em SSES são reduzidas para terminarem em SS, palavras terminadas em IES são reduzidas para terminarem em I.

Além disso, a técnica de Lematização também foi utilizada. Lematização refere-se ao problema de reduzir as palavras à sua forma base, como por exemplo, o verbo conjugado "is" para "be", "cars" para "car".

Também foi utilizada a técnica de feature selection, de forma a reduzir a dimensionalidade dos dados (e assim, diminuir o tempo de execução do algoritmo e reduzir overfitting). Para isso, utilizei o método SelectKBest com score function chi2 ($\chi^2$). Esse método mantém as K características com maior score, obtido através de uma função especificada. Nesse caso, mantém as K características com maior valor obtido no teste de $\chi^2$, que, por medir dependência entre variáveis, em geral desconsidera características que possuem alta probabilidade de serem independentes em relação às classes e dessa forma são irrelevantes para a classificação (como algumas stop words).

Obs.: Foi interessante notar que o desempenho do algoritmo no kaggle aumentou significativamente quando mantive as stop words como features e utilizei a técnica de feature selection. Uma hipótese do motivo pelo qual isso aconteceu é que existem stop words que devem ser relevantes para classificação, como talvez os termos but e not.
