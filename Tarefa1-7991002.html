from nltk.stem import WordNetLemmatizer

class LemmaTokenizer(object):
    def __init__(self):
        self.wnl = WordNetLemmatizer()
    def __call__(self, doc):
        return [self.wnl.lemmatize(t) for t in word_tokenize(doc)]

III - Descrição das técnicas utilizadas¶

A primeira tcnica utilizada para realizar a tarefa foi validao cruzada, utilizando o mtodo holdout, que consiste em dividir os dados de treinamento entre dois conjuntos disjuntos: um de treino e um de validao. Isso foi feito atravs do mtodo train_test_split, com a proporo de dados de treino sendo 2/3. Assim, os parmetros do modelo so estimados no conjunto de treino, e o erro de predio obtido a partir dos dados do conjunto de validao. Dessa forma possvel avaliar a capacidade de generalizao do modelo. Alm disso, o conjunto de validao foi utilizado para reduzir a dimensionalidade dos dados (iterando o nmero de features, avaliou-se qual produzia maior acurcia). Nesse tarefa, foram considerados como 'dados' apenas os textos dos tweets.

Em seguida, o modelo de saco de palavras foi utilizado para representar os dados. Nesse modelo, no consideramos a posio nem a ordem dos termos. Assim, atravs da classe CountVectorizer, o texto representado por um vetor em que cada dimenso corresponde a um token, e o valor da dimenso equivale quantidade de ocorrncias do termo no texto. Uma forma de representao alternativa utilizada na realizao da tarefa foi obtida atravs da classe TfidfVectorizer. A diferena nesse caso que o valor de cada dimenso equivale ao tf-idf do termo no tweet, que obtido atravs da frmula $tf * idf$ (ou por vezes $tf * (idf+1)$), em que tf a frequncia (nmero de ocorrncias) do termo no tweet idf o inverso da frequncia do documento (inverso do nmero de tweets em que o termo ocorre).

Depois, foram utilizados alguns classificadores lineares e avaliamos a acurcia de cada um sobre o conjunto de treino e sobre o de validao. Os classificadores utilizados foram:

1) Perceptron: O algoritmo do Perceptron define uma hiptese final $g(x) = sign(w^tx)$. Ele basicamente inicializa um vetor de pesos $w(0)$ arbitrrio, e para cada iterao $t \ge 0$, escolhe qualquer dado $(x(t), y(t))$ cuja classificao esteja errada (ou seja, $y(t) \ne sign(w(t)^tx(t))$) e atualiza $w(t+1) = w(t) + y(t)x(t)$. O algoritmo para quando no h mais dados mal classificados. Esse algoritmo sempre termina se os dados so linearmente separveis.

2) Regresso Linear: Basicamente tenta achar a hiptese $h$ que minimiza o erro quadrtico entre $h(x)$ e $y$. Para isso, constri a matriz X e o vetor y com os dados $(x_1,y_1), (x_2,y_2), \ldots, (x_N,y_N)$ (em que cada $x_i$ um vetor de dimenso $d$ e $y_i$ corresponde sua classe), e calcula a pseudo-inversa de X: $X^\dagger = (X^TX)^{-1}X^T$. A partir disso, retorna $w = X^\dagger y$.

3) Regresso Logstica: Basicamente, tenta atualiza vetor de pesos $w$ de forma a minimizar $\dfrac{1}{N}\sum_{n=1}^{N}\ln(\dfrac{1}{\theta({y_nw^Tx_n})})$, em que $\theta$ uma funo logstica (como por exemplo, $\theta(s) = \frac{e^s}{1+e^s}$). Para isso, pode-se utilizar o algoritmo de Gradiente Descendente, que basicamente tenta "descer" pela surperfcie que se deseja minimizar at encontrar um ponto mnimo.

Uma outra tcnica que foi abordada nessa tarefa foi a de Stemming, que consiste em uma heurstica que corta o fim das palavras em uma tentativa de obter sua raiz. Essa heurstica baseada em algumas regras, como por exemplo, palavras terminadas em SSES so reduzidas para terminarem em SS, palavras terminadas em IES so reduzidas para terminarem em I.

Alm disso, a tcnica de Lematizao tambm foi utilizada. Lematizao refere-se ao problema de reduzir as palavras sua forma base, como por exemplo, o verbo conjugado "is" para "be", "cars" para "car".

Tambm foi utilizada a tcnica de feature selection, de forma a reduzir a dimensionalidade dos dados (e assim, diminuir o tempo de execuo do algoritmo e reduzir overfitting). Para isso, utilizei o mtodo SelectKBest com score function chi2 ($\chi^2$). Esse mtodo mantm as K caractersticas com maior score, obtido atravs de uma funo especificada. Nesse caso, mantm as K caractersticas com maior valor obtido no teste de $\chi^2$, que, por medir dependncia entre variveis, em geral desconsidera caractersticas que possuem alta probabilidade de serem independentes em relao s classes e dessa forma so irrelevantes para a classificao (como algumas stop words).

Obs.: Foi interessante notar que o desempenho do algoritmo no kaggle aumentou significativamente quando mantive as stop words como features e utilizei a tcnica de feature selection. Uma hiptese do motivo pelo qual isso aconteceu que existem stop words que devem ser relevantes para classificao, como talvez os termos but e not.
